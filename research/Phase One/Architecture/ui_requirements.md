# Phase One — UI Requirements

## Overview

This document defines the frontend user interface requirements for Phase One. All views are read-only (monitoring only, no control actions).

---

## Status: In Progress

---

## Design System

### Component Library

- **Shadcn/ui** — primary component library
- **Shadcn Charts** — built on Recharts
- **Shadcn DataTable** — built on TanStack Table
- **Tailwind CSS** — utility styling

### Theme

- **Dark mode only**
- **Style:** Lyra (Shadcn preset)
- **Base colors:** Zinc and Stone (user-switchable via header toggle)
- **Font:** Noto Sans

### Design Principles

- Simple and minimalistic
- Block / Grid-based layouts where appropriate
- All components built as reusable
- Desktop-first (90% web usage expected)
- Responsive but mobile is not priority
- Loading skeletons for async states (Shadcn Skeleton)

---

## General Requirements

- Responsive design (desktop primary, tablet/mobile supported)
- Dark theme only with Zinc/Stone color toggle
- Clear visual hierarchy
- Real-time data updates (pending WebSocket vs Polling decision)
- Loading skeleton states for async data
- Error states with retry options
- Empty states for tables/lists with no data

---

## Layouts

### Public Layout

- **Purpose:** Unauthenticated pages (Login, Signup)
- **Structure:** Simple centered content, no navigation
- **Pages:** `/login`, `/signup`

### Protected Layout

- **Purpose:** Authenticated pages
- **Structure:**
  - **Header** — logo, theme toggle, notification bell (with badge), user menu
  - **Collapsible Sidebar** — main navigation
  - **Main Content Area** — page content
  - **Footer** — minimal info if needed

## Reusable Page Pattern (PageShell)

Most data pages follow a consistent structure:

```
PageShell
├── PageHeader
│   ├── Title
│   ├── Description (optional)
│   └── Actions (buttons, optional)
├── FilterBar
│   ├── Search input
│   ├── Filter dropdowns (dynamic)
│   └── Action buttons (dynamic)
└── DataTable
    ├── Sortable columns
    ├── Pagination
    └── Row actions (if needed)
```

Components are dynamic — filters, buttons, and columns configured per page.

## Pages / Views

### 1. Login Page

**Route:** `/login`
**Layout:** Public

**Elements:**

- Email input
- Password input
- Login button
- "Create account" link → `/signup`
- Error message display

**Behavior:**

- Redirect to Dashboard on success
- Show error on invalid credentials

---

### 2. Signup Page

**Route:** `/signup`
**Layout:** Public

**Elements:**

- Email input
- Access code input (provided by admin)
- Password input
- Confirm password input
- Signup button
- "Already have an account?" link → `/login`
- Error message display

**Behavior:**

- Validate access code matches email (admin-generated)
- Create user account on valid code
- Redirect to Login on success
- Show error on invalid/expired code or mismatched email

**Note:** Access codes are generated by admins for specific email addresses. This allows controlled public registration.

---

### 3. Dashboard (Home)

**Route:** `/`
**Layout:** Protected

**Purpose:** High-level server overview using block/grid layout

**Design:** Grid of cards, responsive columns

**Cards:**

#### Server Status Card

- Hostname
- Uptime
- OS version
- Last updated timestamp

#### CPU Card

- Current usage (gauge or percentage)
- Historical chart (configurable period)

#### Memory Card

- Used / Total (e.g., "2.1 GB / 4 GB")
- Percentage bar
- Historical chart

#### Network Card

- Current In/Out rates
- Historical chart

#### Disk Usage Card

- Used / Total for primary disk
- Percentage bar
- (No historical chart per requirements)

#### Storage / Mounts Card

- List of mounted storage (internal, external, NAS)
- Per mount: path, type badge, used/total, status
- Compact view (expandable if many mounts)

#### Quick Stats Card

- Running processes count
- Active services count
- Docker containers count
- Active Node.js servers count

### 10. Users Page (Admin only)

**Route:** `/users`
**Layout:** Protected
**Pattern:** PageShell

**Elements:**

- Search by username/email

  - Email

- Code expiry info

---

### 11. Profile Page

**Route:** `/profile`
**Layout:** Protected

**Sections:**

#### Personal Information

- Display name (editable)
- Email (read-only)
- Avatar upload (optional, future)

#### Security

- Change password form

#### Settings (if needed)

- Notification preferences
- Other user-specific settings

---

### 12. Error Pages

#### 404 — Not Found

**Route:** `*` (catch-all)
**Layout:** Minimal (no sidebar)

**Elements:**

- "Page not found" message
- "Go to Dashboard" button

#### Error — Something Went Wrong

**Layout:** Minimal
**Trigger:** Error boundary catch

**Elements:**

- "Something went wrong" message
- "Try again" button
- "Go to Dashboard" button
